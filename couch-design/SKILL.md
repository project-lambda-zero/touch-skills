## How to honour this brief in practice

These are the operational rules the brief generates. They translate the feeling into commitments that pass mechanical review.

### 1. The childhood-memory test

Before shipping any design element ; color, type, layout, motion, micro-detail ; ask: *does this element have the shape of something the visitor could remember from their own life?*

Things that pass: the way an early Mac System dialog sat on the screen. The blue of a faded Tron VHS sleeve. The corner of a peach-gum wrapper. The off-register cyan of risograph zines. The seam between two pages of a science textbook. The way a Trapper Keeper grid felt under your hand. The sticker album with the foil border. The taste of paper at the back of a library book. The flicker of an old CRT before the picture stabilised. The 1990s shareware splash screen that came up after you double-clicked an `.EXE` on Windows 98.

Things that fail: the third gradient mesh in a row. A glassmorphism card. A `text-7xl tracking-tighter leading-none` hero. Any element that exists because shadcn ships it that way.

The element doesn't have to literally reproduce a memory ; it has to share that memory's *structural texture*. A panel might not be a Trapper Keeper, but if it has the same "sectioned, ringed, intentional" feel, it qualifies.

### 2. Every element earns its place (single-goal rule)

Each element on the page exists to do ONE thing. If you cannot name in a single sentence what that thing is, the element is filler. Filler is banned. Examples:

- An "eyebrow" label above every section ; banned, that's a templated reflex, no unique goal.
- A status dot before a button label ; only if the dot actually communicates "live"; otherwise it's filler.
- A second CTA that says the same thing as the first ; banned.
- A footer "v0.6.2" tag ; only if there IS a version anyone cares about.
- A "ESTD. 2024 · LISBON" mono strip ; banned unless the place actually matters to the brief.

### 3. Nothing repeats

If two elements share the same visual shape, type treatment, layout family, or color scheme, one of them must change or be cut. The brief is non-negotiable here ; "nothing is repeated, everything is unique" appears twice in the brief and counts as a hard rule.

This applies to:
- Section layouts (no two "3-col cards in a row")
- Type pairings (no two sections both opening with the same eyebrow + headline + body shape)
- Color rhythm (don't put the same accent block twice in a row)
- Motion (don't reveal everything with the same fade-up)
- Voice (don't repeat the same phrase across hero, sub, CTA, footer)

### 4. The neurodivergent pattern-detection rule

The operator detects AI defaults instantly. This is a real constraint, not a stylistic preference. When choosing between:

- The popular pick (`Inter`, `text-zinc-950`, glassmorphism, dark mode with neon accent, three feature cards, "Trusted by" logo strip, hero-with-asset-on-right, etc.)
- An unpopular pick that has a real reason

The unpopular pick is correct unless the popular one is the only thing that fits a real constraint. If you find yourself reaching for a popular pick, stop, ask why, and pick the second-best option that has a genuine reason. There is no aesthetic high ground that is widely-accepted enough to satisfy this brief.

### 5. The respect rule

Respect the reader. Concrete commitments:

- WCAG AA contrast everywhere (operator may not consciously enforce this; you must)
- Real punctuation (no em-dash as a stylistic move ; uses semi-colon, hyphen, comma, or full clause)
- Honest data (no `99%` made-up numbers, no "Trusted by" without real logos, no testimonials with fake names)
- Reading-comfortable line length (50-75 chars, never wider for body)
- Real font choices for fluency (no decorative type for body, ever)
- Mobile not as an afterthought
- Reduced-motion respected
- No interception of standard browser behaviour (no custom cursor, no scroll hijack, no smooth-scroll override on the whole page)

### 6. Bond, don't perform

The site should feel like the operator, not like a marketing department. Concretely:

- Voice is first-person where it makes sense, not "we" boilerplate
- Copy admits the gaps (what the project does NOT do, what is still rough, what is opinionated)
- Numbers come with provenance
- The footer mentions the maintainer the way a small zine mentions its editor

Two words banned forever for this operator:

- `load-bearing` as an analytical adjective ; it is filler jargon.
  Name the actual function the thing performs ("the chrome shows the
  panel boundary"; "the cutout is the brand mark") instead.
- `internal sec` ; meaningless audience label invented during a 2026
  session. The page must not invent customer / persona labels;
  describe the platform's mechanism, not a synthetic buyer profile.

### 7. Search before committing

The brief says: *"i want you to intelligently search through them, enhance with the deep web research them, then we will create this design."* This is a literal directive. Before locking an aesthetic, run real web search on:

- Unusual personal sites (not "top 50 SaaS landing pages")
- Designer portfolios from people the audience would recognise as having taste
- Editorial / book design ; especially small press, art books, sci-fi paperback covers
- Old computing aesthetics ; Amiga workbench, Atari ST GEM, early Mac, Plan 9, BeOS, OS/2 Warp, NeXT
- Risograph zines, comic book grids, mid-century book endpapers
- Concrete poetry, type specimens, manifesto posters
- Recent winners of awwwards or Site Inspire ONLY if filtering out anything Stripe/Linear/Vercel-shaped

Use the results as concrete reference points the operator can react to, not as a backdrop you absorb silently.

### 8. The 02:00 am clause

The brief was written by a broken heart at 02:00 am. The agent does not psychoanalyse, does not pity, does not perform empathy theatre. The agent sits on the couch and does the work. The honour is in shipping something the operator will look at and recognise.

## What this skill explicitly does NOT do

- It does not replace `frontend-design`, `design-taste-frontend`, `ui-ux-pro-max`, or other technical-craft skills. Those load alongside this one when relevant. This skill is the *commitment layer*; those are the *technique layers*.
- It does not auto-select a direction. The operator is the art director; the agent's job is to surface non-generic options and execute the chosen one perfectly.
- It does not relax for "quick" or "MVP" requests. The 02:00 am brief is unconditional. There is no fast path that bypasses it.
- It does not invoke this voice for other operators or other repos that haven't asked for it. This is the operator's personal aesthetic contract.
